# Sentry Sound Academy SLB Review Matrix V1

## Purpose

This document defines the mandatory review matrix for Academy SLBs.

An SLB is the smallest controlled competency execution block.

No SLB may proceed to lesson, assessment, remediation, or runtime design unless it passes this review.

---

# Core Rule

Every SLB must be:
- teachable
- assessable
- measurable
- evidence-capable
- remediation-capable
- runtime-compatible
- standards-ready
- operationally relevant

---

# Section 1 — SLB Identity

Required:
- SLB ID
- SLB Name
- Module ID
- Module Name
- Academy Layer
- Competency Level
- SLB Type
- Version
- Status
- Reviewer
- Review Date

---

# Section 2 — Competency Review

Validation:
- Is the competency small enough?
- Is it observable?
- Is it measurable?
- Can the learner demonstrate it?
- Can failure be identified clearly?
- Can competency state be tracked?

Approval status:
- approved
- revision_required
- rejected

---

# Section 3 — Outcome Review

Validation:
- Is the primary outcome clear?
- Are supporting outcomes relevant?
- Are success criteria measurable?
- Are failure indicators clear?
- Is the outcome linked to module purpose?

Approval status:
- approved
- revision_required
- rejected

---

# Section 4 — Evidence Review

Validation:
- What evidence proves this SLB?
- Is evidence realistic?
- Is evidence verifiable?
- Can evidence be stored or referenced?
- Can evidence support future moderation/audit?

Approval status:
- approved
- revision_required
- rejected

---

# Section 5 — Assessment Review

Validation:
- Can the SLB be assessed directly?
- Is assessment practical?
- Is pass/fail logic clear?
- Can reassessment be done?
- Can assessment trigger remediation?

Approval status:
- approved
- revision_required
- rejected

---

# Section 6 — Remediation Review

Validation:
- Can weak competency be remediated?
- Is the remediation target clear?
- Can remediation be routed to a smaller concept/workflow?
- Can reassessment confirm improvement?

Approval status:
- approved
- revision_required
- rejected

---

# Section 7 — Platform Alignment Review

Use when applicable.

Validation:
- Does this SLB align to real Sentry Sound Platform logic?
- Does it reference correct platform entities?
- Does it reference correct metadata/workflow/status logic?
- Does it avoid fake platform processes?

Approval status:
- approved
- revision_required
- rejected
- not_applicable

---

# Section 8 — Standards Readiness Review

Validation:
- Is the SLB outcomes-based?
- Is it competency-based?
- Can it support future NQF mapping?
- Can it support credit/notional-hours estimation?
- Can it support future certification evidence?

Approval status:
- approved
- revision_required
- rejected

---

# Section 9 — Runtime Compatibility Review

Validation:
- Can learner activity be tracked?
- Can events be generated?
- Can competency state be updated?
- Can evidence submission be linked?
- Can remediation triggers be created?
- Can reporting consume this SLB?

Approval status:
- approved
- revision_required
- rejected

---

# Section 10 — Governance Review

Validation:
- Is the SLB version-controlled?
- Is review status clear?
- Is approval needed before use?
- Can changes be audited?
- Can retired SLBs remain historically traceable?

Approval status:
- approved
- revision_required
- rejected

---

# Final Decision

Allowed statuses:
- approved_for_lesson_design
- approved_for_assessment_design
- approved_for_runtime_design_later
- revision_required
- rejected
- archived

---
