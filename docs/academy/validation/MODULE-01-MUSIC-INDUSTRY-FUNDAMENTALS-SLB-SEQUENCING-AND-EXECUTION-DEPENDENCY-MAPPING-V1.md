# Core Principle

An SLB is not merely a content block.

An SLB is a competency-state transition unit.

An SLB may later:
- teach
- assess
- validate
- remediate
- generate evidence
- generate telemetry
- trigger orchestration
- influence learner progression

---

# Runtime Philosophy Alignment

The academy is AI-mediated.

Learning is not primarily lecturer-driven.

The future runtime should eventually:
- guide competency execution
- measure competency states
- detect weaknesses
- route remediation
- adapt progression
- track evidence
- produce telemetry
- orchestrate learner progression

This sequencing document prepares for that future runtime direction.

---

# Sequencing Logic Principles

SLB sequencing should support:

1. foundational terminology before workflow logic
2. ecosystem awareness before operational detail
3. ownership awareness before royalty awareness
4. platform awareness before platform operations
5. competency progression before assessment escalation
