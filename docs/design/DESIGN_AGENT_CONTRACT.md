# Design Agent Contract

## Status

Future governance/reference guidance only.

This document does not activate a Design Agent workflow, freeze frontend work, require Figma-first execution, or introduce design-system bureaucracy.

## Purpose

The future Design Agent lane exists to help Sentry Sound retain reusable design intelligence over time:

- reusable operational UX patterns
- reusable visual language
- reusable section and card structures
- reusable interaction and state patterns
- reusable workflow assets
- retrieval-compatible design memory

The Design Agent is subordinate to Sentry Sound operational contracts and backend source-of-truth rules.

## Authority Boundaries

Backend/source-of-truth remains authoritative.

Frontend remains an interpretation and presentation layer.

The UI may:

- present workflow state
- guide user action
- collect input
- reuse approved patterns
- improve product clarity

The UI may not:

- own workflow truth
- invent approval state
- override backend readiness/evidence logic
- expose private data as public-safe
- treat visual artifacts as source-of-truth

## Codex Relationship

Codex remains the active implementation agent.

Codex should continue:

- practical frontend implementation
- UX refinement
- reusable component and pattern work
- asset organization
- AI-assisted visual direction workflows
- product-first iteration

Future Design Agent guidance should help Codex retrieve and adapt prior design intelligence. It should not slow useful product delivery.

## Figma / Visual Source Position

Figma may become a future editing or review surface.

Figma is not automatically the source of truth.

Assets are assets. Reusable patterns, semantic components, workflow structures, and operational contracts remain more important than any single design tool.

## Compatibility Goal

Build practical product UX now while remaining compatible with future governance maturity.
