# Canonical Entity Map

## Purpose

This map declares the current and target ownership of Sentry Sound Platform entities. It prevents future backend work from attaching features to the wrong model.

Status labels:

- active canonical seed: used by the current Works/Songs UX and approved as the seed for canonical alignment.
- future canonical: target entity not fully active yet.
- legacy/parallel: exists, but must not be treated as active product truth until aligned.
- test-only: supports TEST/V1 surfaces only.

Finance V2 field model reference:

The canonical future Finance V2 field architecture is defined in `docs/platform/FINANCE-V2-FIELD-MODEL.md`. Finance transactions remain the accounting/money anchor; needs/business cases, budgets, allocations, deliverables, approvals, evidence, and ROI/value results are future linked entities or workflow-derived records.

Finance V2 Phase 1 reference:

Phase 1 is actuals + commitments + reserve awareness. Budgets, forecasts, ROI intelligence, and AI financial coaching are later phases.

## Current Lock

The active Works/Songs UX currently uses the lowercase operational seed. Any new backend work touching work identity, identifiers, submissions, or readiness must declare whether it reads or writes this canonical seed, a future canonical entity, or a legacy/parallel model.
