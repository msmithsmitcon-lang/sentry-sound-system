# Sentry Sound Academy Canonical LLM Generation Approach

## Purpose

This document locks the approach for future course and content development.

The academy must not manually reinvent every module, lesson, SLB, assessment, remediation flow, or evidence structure from scratch.

Instead, the academy will use a controlled canonical LLM generation prompt/template.

---

# Core Principle

One master generation structure should be created and reused across academy modules.

Only controlled inputs should change, such as:

- module name
- module level
- competency level
- learner profile
- industry body alignment
- platform/backend alignment
- intended qualification or certification pathway
- target evidence requirements

---

# Why This Matters

This prevents:
- inconsistent curriculum structures
- disconnected lesson formats
- random assessment design
- misalignment with Sentry Sound Platform logic
- misalignment with competency standards
- unnecessary manual work
- uncontrolled LLM hallucination
- curriculum drift

---

# Future Master Prompt Must Generate

For each module, the canonical LLM prompt must be able to generate:

1. Module overview
2. Purpose statement
3. Target learner profile
4. Competency level
5. Industry relevance
6. Outcomes
7. Assessment criteria
8. SLB-style competency blocks
9. Lesson structure
10. Practical exercises
11. Platform alignment notes
12. Metadata requirements
13. Evidence requirements
14. Remediation logic
15. Rubrics
16. Facilitator notes
17. Learner notes
18. Verification checklist
19. Certification/NQF alignment placeholder
20. Runtime execution readiness notes

---

# Controlled Generation Rule

The LLM must not invent uncontrolled structures.

It must work from:
- the academy module list
- the competency framework
- the canonical module blueprint
- Sentry Sound Platform backend logic where relevant
- industry standards alignment notes
- assessment and evidence rules
- future SAQA/SETA/NQF mapping documents
