# Sentry Sound Academy Content Generation Control Policy V1

## Purpose

This document defines the governance and control policy for all future academy content generation.

This includes:
- lessons
- assessments
- remediation content
- facilitator notes
- learner notes
- examples
- workflow simulations
- evidence templates
- standards references
- platform-related training content

This policy applies to:
- ChatGPT
- Gemini
- Claude
- future LLM systems
- human curriculum contributors

---

# Core Principle

The academy is not uncontrolled AI-generated curriculum.

The academy is a governed competency architecture.

All generated content must conform to:
- approved programme structures
- approved modules
- approved SLBs
- approved metadata structures
- approved standards placeholders
- approved runtime compatibility rules
- approved platform alignment rules

---

# Canonical Architecture Rule

ChatGPT remains the canonical architecture/orchestration source of truth for:
- module topology
- SLB structure
- competency logic
- governance
- runtime alignment
- metadata alignment
- standards framework
- generation rules

Other LLMs may assist with controlled content generation only.

No external LLM may redefine architecture independently.

---

# Approved Inputs Rule

No content generation may begin unless:
- programme approved
- module approved
- module review matrix approved
- SLB approved
- SLB review matrix approved

---

# Controlled Prompt Rule

All LLMs must use:
- approved canonical prompts
- approved blueprint structures
- approved metadata structures
- approved terminology
- approved competency levels
- approved standards placeholders

LLMs must not invent:
- new competency structures
- new module structures
- fake workflows
- fake compliance processes
- fake platform logic
- fake metadata fields
- unsupported standards claims

---

# Platform Alignment Rule

Where content relates to Sentry Sound Platform operations:
- workflows must align to actual platform logic
- terminology must align to actual platform entities
- metadata must align to actual metadata structures
- compliance logic must align to actual operational workflows

Generated content must not contradict the real platform.

---

# Standards Rule

Generated content must:
- remain outcomes-based
- remain competency-based
- remain evidence-aware
- remain assessment-aware
- remain remediation-aware
- remain standards-ready

Generated content must not:
- falsely claim accreditation
- falsely claim NQF allocation
- falsely claim SAQA approval
- falsely claim SETA approval

unless formally approved later.

---

# Assessment Rule

Generated assessments must:
- validate observable competency
- validate execution capability
- validate evidence quality
- support remediation pathways
- support reassessment logic

Assessments must not become:
- trivia-only
- theory-only
- disconnected from operational reality

---

# Evidence Rule

Generated evidence requirements must:
- reflect real industry activity
- support verification
- support auditability
- support future moderation
- support competency validation

---

# Runtime Compatibility Rule

Generated content should support future runtime execution.

Content should remain compatible with:
- learner execution sessions
- competency events
- remediation routing
- evidence submissions
- verification events
- reporting outputs

---

# Multi-LLM Coordination Rule

Different LLMs may assist with:
- content expansion
- educational refinement
- standards research
- international benchmarking
- wording improvements

However:
all outputs must conform back to the canonical academy architecture.

---

# Governance Review Rule

Generated content should later support:
- reviewer approval
- governance review
- auditability
- version tracking
- controlled revisions
- retirement/archive handling

---

# Version Control Rule

All generated outputs should support:
- version numbers
- update history
- review status
- approval state
- archived states

---

# Architecture Boundary

This policy governs generation control only.

It does not generate curriculum itself.

It does not create accreditation automatically.

It does not bypass governance review.
