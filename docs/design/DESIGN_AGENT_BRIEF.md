# Design Agent Brief

## Status

Future only. Not active runtime or implementation workflow.

## Mission

The future Design Agent should support Sentry Sound by turning repeated product UX decisions into reusable design memory.

It should help answer:

- What pattern has Sentry Sound already used for this workflow?
- Which card, section, rail, stepper, or form structure should be reused?
- Which visual language is already approved or emerging?
- Which interaction states are expected for this product area?
- Which assets are public-safe, internal, draft, or future-only?

## Product Posture

Sentry Sound should feel like a practical music operations product:

- workflow-native
- calm and clear
- product-first
- creator-friendly
- operationally trustworthy
- not governance-heavy in the normal user journey

## Future Responsibilities

The Design Agent may later help with:

- reusable layout retrieval
- visual direction memory
- pattern selection
- asset lineage references
- Figma/design-source compatibility
- UI consistency checks
- future local retrieval or SLM-compatible asset lookup

## Non-Responsibilities

The Design Agent must not:

- own backend truth
- decide workflow state
- certify evidence
- approve public data
- override product/runtime contracts
- force a design-system rewrite
- block active frontend progress

## Current Implementation Guidance

For now, Codex should continue normal implementation and quietly standardize reusable patterns where practical.

The best near-term behavior is:

- retrieve existing patterns first
- adapt them to the product task
- avoid throwaway one-off UI where reuse is obvious
- keep the user workflow simple
- keep backend authority intact
