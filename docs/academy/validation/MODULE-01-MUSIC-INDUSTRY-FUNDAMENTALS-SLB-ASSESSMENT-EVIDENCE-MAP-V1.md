# Module 01 — Music Industry Fundamentals
# SLB-to-Assessment & Evidence Map V1

## Purpose

This document maps each approved SLB to:
- assessment type
- evidence type
- competency signal
- remediation trigger
- AI delivery implication

This is not final assessment creation yet.

It defines what kind of evidence and assessment logic each SLB should support.

---

# Core Principle

Every SLB must be assessable.

Every assessment must produce a competency signal.

Every weak competency signal must be capable of triggering remediation.

Because the academy is AI-presented, assessment and evidence should support AI-guided progression rather than passive quiz completion.

# Evidence Strength Levels

## Lightweight Evidence
Used for foundation-level awareness.

Examples:
- short explanation
- matching response
- simple map
- checklist
- terminology answer

## Structured Evidence
Used where learner must organise relationships.

Examples:
- ecosystem map
- workflow diagram
- lifecycle sequence
- role/entity map
- ownership comparison

## Future Strong Evidence
Reserved for later operational/professional modules.

Examples:
- registration pack
- metadata pack
- royalty calculation
- compliance report
- contract review
- audit trail review

---

# AI Delivery Implications

The AI presenter should later:

- ask diagnostic questions
- detect incorrect terminology
- test role/entity understanding
- check workflow sequencing
- request simple explanations
- compare learner responses against competency criteria
- trigger remediation where weak signals appear
- avoid advancing learners blindly

---

# Remediation Logic

Remediation should be targeted.

Examples:

- terminology failure → route to SLB-01.14
- role confusion → route to SLB-01.02
- lifecycle sequencing failure → route to SLB-01.04, SLB-01.05, or SLB-01.06
- ownership confusion → route to SLB-01.07 and SLB-01.08
- metadata misunderstanding → route to SLB-01.12
- royalty-flow weakness → route to SLB-01.09
