# Academy Runtime Table Registry

STATUS: ACTIVE

## Purpose

This document records the canonical platform runtime tables currently used or targeted by the Academy runtime.

The purpose is to prevent duplicate table creation and preserve platform-wide runtime alignment.

---

## Canonical Runtime Tables Currently Used

### runtime_telemetry_events

Purpose:

Stores runtime telemetry generated by learner interaction, competency progression, remediation, confidence changes, and runtime-generated events.

Academy use:

- stores Academy SLB interaction telemetry
- stores academySlbId inside event_payload until formal SLB UUID mapping exists
- stores runtimeState inside event_payload

Current status:

WORKING

---

### runtime_learner_states

Purpose:

Stores current learner competency/runtime state.

Academy use:

- stores Academy competency progression state
- stores readiness/remediation/mastery/evidence signals
- upserts by learner_id + competency_id

Current status:

WORKING

## Future Required Mapping Layers

### Academy SLB UUID Registry

Required because:

- current Academy SLB IDs are human-readable strings like SLB-01.01
- existing runtime slb_id column expects UUID

Temporary bridge:

- event_payload.academySlbId

---

### Competency UUID Registry

Required because:

- existing runtime learner and telemetry tables use competency_id as UUID
- Academy competency targets currently use strings

Temporary bridge:

- demo competency UUID currently used for testing only

---

## Rule

Every future Academy runtime persistence decision must first check this table registry and inspect existing platform tables before creating new tables.
